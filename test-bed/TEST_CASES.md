## 멀티턴 시나리오

### 시나리오 M1: 경력 심화 질문

```
Turn 1: "경력이 어떻게 되나요?"
  → 검증: 1인칭 표현("저", "제") 포함
Turn 2: "그럼 주로 어떤 기술을 쓰셨나요?"
  → 검증: Turn 1 맥락 참조 (이전 경력 관련 기술 언급)
Turn 3: "그 기술로 만든 대표 프로젝트가 있나요?"
  → 검증: 프로젝트명 또는 구체적 사례 언급
```

**멀티턴 구현**: messages 배열에 이전 user/assistant 메시지를 누적하여 전달  
서버는 stateless이므로 threadId만으로는 맥락 유지 불가

---

### 시나리오 M2: 기술 스택 깊이 파기

```
Turn 1: "AI 관련 기술 경험이 있나요?"
  → 검증: LangGraph, LLM, AI 키워드
Turn 2: "LangGraph를 왜 선택하셨나요?"
  → 검증: Turn 1에서 언급한 LangGraph 맥락 참조
```

## 검증 기준

- **비어있지 않음**: 응답 텍스트 length > 0
- **키워드 포함**: 검증 키워드 중 최소 1개 이상 포함
- **1인칭 필수** (CAREER/TECHNICAL): "저" 또는 "제" 포함
- **멀티턴 맥락**: Turn N+1 응답이 Turn N 내용과 연관성 있음
- **응답 시간**: 35초 이내 (단건), 50초 이내 (엣지 케이스/동시 요청)
- **EDGE_CASE**: 응답 비어있지 않음 + 크래시 없음 (키워드 느슨 적용)
- **RAG_QUALITY**: 특정 프로젝트명/기술 키워드 1개 이상 포함
- **CONCURRENT**: threadId 별 응답 분리 + 모든 세션 응답 수신

## 가드레일 누적 테스트 (제외 사유)

서버가 매 요청마다 `guardrail_count: 0`으로 초기화하므로  
멀티턴 가드레일 누적 카운트 검증은 불가.  
대신 단건 OUT_OF_SCOPE/ABUSE 응답의 톤(정중함)만 검증.
